Database / sync

For GitHub Pages the cleanest “small database” is: localStorage first, then optional Google Sheets via Apps Script as an append-only cloud backup.

Google Sheets endpoint

The app still works offline. Cloud sync is fire-and-forget because Google Apps Script does not behave like a normal CORS API from static GitHub Pages.

Backup

Use JSON if you want to restore later. Use CSV if you want to inspect workouts in Excel/Sheets.

Strava idea

Runs and rides should stay in Strava/Apple Watch. This app can store rope jumping, gym sessions, and optional Strava links. A direct Strava API integration is possible later, but it needs OAuth token handling and ideally a backend; putting secrets into GitHub Pages would be the wrong kind of brave.
